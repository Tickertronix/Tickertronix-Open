# Contributing to Tickertronix-Open

Thank you for your interest in contributing! This project welcomes contributions from the community.

## Ways to Contribute

- **Bug Reports**: Open an issue with reproduction steps
- **Feature Requests**: Describe your use case and proposed solution
- **Code Contributions**: Submit pull requests with improvements
- **Documentation**: Help improve setup guides and troubleshooting
- **Hardware Designs**: Share 3D models, wiring diagrams, or new display implementations

## Development Setup

1. Fork the repository
2. Clone your fork
3. Create a feature branch: `git checkout -b feature/my-feature`
4. Make your changes
5. Test thoroughly
6. Submit a pull request

## Code Style

- **Python**: Follow PEP 8
- **CircuitPython**: Follow Adafruit style guide
- **Arduino/C++**: Follow Arduino style conventions
- **Documentation**: Use clear, concise language with examples

## Testing

- Test all changes on actual hardware when possible
- Include test cases for API changes
- Verify documentation accuracy

## Pull Request Process

1. Update documentation for any user-facing changes
2. Add your changes to CHANGELOG.md (if applicable)
3. Ensure all tests pass
4. Request review from maintainers
5. Address review feedback
